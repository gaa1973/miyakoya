# MIYAKOYA Project Specification

## 1. プロジェクトの目的 (Project Goals)
本プロジェクトの最重要目標は以下の通りです。

*   **最もエレガントなランディングページ（LP）の作成**:
    *   見る人を魅了する、圧倒的な美しさと品格を備えたデザインを実現する。
    *   「伝統と現代」「夢と現実」「舞台上の輝き」といったブランドコンセプトを体現する。
*   **年商1億円の達成**:
    *   上記のLPを通じて、ブランド価値を最大化し、購買意欲を高めることで、年間売上1億円を達成する。

## 2. 開発・運用ルール (Development & Rules)
開発および修正において、以下のルールを**絶対に遵守**すること。

*   **Chrome DevToolsによる検証の徹底**:
    *   ソースコードを修正した際は、**必ず** Google Chrome のデベロッパーツール（DevTools）を使用して、修正箇所の表示確認および動作確認を行うこと。
    *   レスポンシブ表示（モバイル、タブレット、デスクトップ）の確認も忘れないこと。
*   **パフォーマンス最適化**:
    *   **重要課題**: TOPページの初回表示速度が遅いため、これを改善し、高速化すること。
    *   画像の最適化、スクリプトの遅延読み込み、キャッシュ戦略などを検討・実施する。

## 3. Webアプリケーション仕様 (Web Application Specs)
### 3.1. デザインガイドライン
*   **コンセプト**: "Modern Tradition" - 伝統と現代の融合。
*   **美的要件**:
    *   **Elegant (優雅)**: バレエの優美さを表現する。
    *   **Premium (高級感)**: ハンドメイドの価値を伝える高品質なビジュアル。
    *   **Dynamic (動的)**: フェードインやパララックス効果を用い、静止画だけではない「生きている」デザインにする。
*   **インタラクション**: ユーザーを飽きさせないマイクロインタラクションやホバーエフェクトを実装する。

### 3.2. 技術スタック (Technical Stack)
*   **Frontend**: HTML5, CSS3 (Vanilla), JavaScript (Vanilla)
*   **Deploy**: Vercel (現状の構成より推測)

### 3.3. コンテンツ要件
*   **Hero Section**: 
    *   "Weaving Dreams into Reality on the Stage"
    *   "指先から生まれる魔法、あなただけの物語を纏う瞬間。"
*   **Concept**: ブランドフィロソフィーの訴求。
*   **Collection**: 
    *   Classical Tutu (クラシックチュチュ)
    *   Tiara (ティアラ)
    *   Accessories (アクセサリー)
*   **Shop & SNS**: 
    *   BASE / Minne への導線。
    *   Instagram への導線。

## 4. SEO & パフォーマンス要件
*   **Core Web Vitals**: 特に **LCP (Largest Contentful Paint)** を重点的に改善する。
*   **SEO**: 
    *   適切な `title`, `meta description` の設定。
    *   OGPタグの設定（SNSシェア対応）。
    *   セマンティックなHTML構造 (`header`, `main`, `section`, `footer` 等の適切な使用)。

## 5. 開発・検証ワークフロー（重要）

AIエージェントは、コードの修正や機能追加を行う際、**必ず以下のサイクルを守って**作業を進めてください。ユーザーへの確認を求める前に、自律的に動作確認を行うことを原則とします。

### 作業ループ
1.  **実装・修正:** 指示に基づきコードを編集する。
2.  **ローカル検証:** 修正後、**必ず**ローカルサーバーを起動（または更新）し、ブラウザで実際の画面を開いて表示や動作を確認する。
3.  **自己修正:** 表示崩れやエラーがあれば、その場で修正し、再度検証する。
4.  **報告:** 正常な動作が確認できて初めて、ユーザーに完了報告を行う。

### 確認時のチェックポイント
*   レイアウトが崩れていないか（PC/スマホ両方で見る）。
*   コンソールにエラー（赤い文字）が出ていないか。
*   実装した機能が意図通りに動いているか。

## 6. その他
*   **ソースコード修正時の注意**: 既存の「エレガントさ」を損なう修正は行わないこと。
