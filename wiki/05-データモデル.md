# 🗄️ 05. データモデル

## 🎨 素材・コンテンツ管理

本プロジェクトは静的サイトであるため、データベースは使用していませんが、画像素材や情報の構成管理のために論理的なデータ構造を持っています。

### 1. アセット管理 (Assets)

画像アセットは `assets/` ディレクトリに集約され、用途に応じて命名されています。

| カテゴリ       | プレフィックス/ファイル名                     | 役割                                 |
| :------------- | :-------------------------------------------- | :----------------------------------- |
| **ロゴ**       | `miyakoya_logo.png`, `miyakoya_sub_logo.png`  | ブランドアイデンティティ             |
| **背景**       | `hero_bg_optimized.jpg`                       | ヒーローセクションのメインビジュアル |
| **カテゴリー** | `collection_tiara.png`, `collection_tutu.png` | コレクション一覧の代表画像           |
| **商品**       | `tutu_lilac.jpg`, `tutu_pink.jpg`             | 個別商品の詳細画像                   |
| **プロセス**   | `making_process.png`                          | 制作風景の紹介                       |

### 2. コンテンツ構造

各セクションのコンテンツ（テキスト、価格等）は HTML の各要素にハードコードされており、以下の構造で管理されています。

- **Service Menu**:
  - `Order Made`: 価格、特徴、ターゲット。
  - `Rental`: 大人/子供別の価格、期間、特徴。
- **Order Flow**:
  - Step 1 (Contact) -> Step 2 (Measurement) -> Step 3 (Creation) -> Step 4 (Delivery)。

### 3. 動的データの扱い

JavaScript (`js/main.js`) は以下の「状態」や「データ」を動的に制御します。

- **Loading State**: 背景画像の読み込み等に合わせた待機時間。
- **Scroll State**: ヘッダーの透過度や、各セクションのフェードアニメーションのトリガー。
- **Navigation State**: モバイルメニューの開閉状態。

## 🔧 データ更新の手順

1.  **画像の追加**: `assets/` に追加し、`optimize_image.py` を使用して最適化を行う。
2.  **文言・価格の変更**: `index.html` または `lookbook.html` の該当するタグ（クラス名がヒント）を編集する。

詳細は [コアコンポーネント](./06-コアコンポーネント.md) を参照してください。
