# 🧩 06. コアコンポーネント

## 🏛️ 主要な UI セクション

`index.html` は以下の独立したセクションで構成されており、それぞれがブランド体験の重要な一部を担っています。

1.  **Hero Section (`.hero`)**:
    - プロジェクトの第一印象を決定するメインビジュアル。
    - `hero_bg_optimized.jpg` を背景に使用し、パララックス効果を適用。
2.  **Features Section (`.features`)**:
    - 「3 つのこだわり」をグリッドレイアウトで表示。
3.  **Concept Section (`.concept`)**:
    - ブランド哲学。画像とテキストが対比されるエレガントなレイアウト。
4.  **Service Guide (`.service-guide`)**:
    - オーダー（Order）とレンタル（Rental）の対比表。
5.  **Order Flow (`.service-flow`)**:
    - 問い合わせからお届けまでの 4 ステップを可視化。
6.  **Collection Section (`.collection`)**:
    - `lookbook.html` への導線となるギャラリー。

## 🧠 JavaScript ロジック (`js/main.js`)

主要な機能のコード解説です。

### 1. ローディング画面の制御

初回訪問時にロゴを表示し、一定時間（またはハッシュがある場合は短縮）でフェードアウトさせます。

```javascript
// js/main.js:3-23
const loadingScreen = document.getElementById("loadingScreen");
if (loadingScreen) {
  let delayTime = 3000;
  if (window.location.hash) {
    delayTime = 2000;
    loadingScreen.style.animation = "fadeOut 1.5s ease 0.5s forwards";
  }
  // ...
}
```

### 2. モバイル CTA の動的切り替え

ユーザーのスクロール位置や、現在表示されているセクション（Order Flow 等）に応じて、下部のボタン（Shop または Contact）を切り替えます。

```javascript
// js/main.js:35-96
const flowSection = document.querySelector(".service-guide");
// IntersectionObserver を使用してセクションの可視性を監視
if (flowSection && mobileCta) {
  const flowObserver = new IntersectionObserver(...);
}
```

### 3. スムーススクロール

ヘッダーの高さ（Offset）を計算し、ページ内リンクを滑らかに移動させます。

```javascript
// js/main.js:121-162
document.querySelectorAll('a[href^="#"]').forEach((anchor) => { ... });
```

### 4. スクロールアニメーション

`.fade-in-up` 等のクラスを持つ要素が画面に入った際に、`visible` クラスを付与してアニメーションを開始します。

詳細は [API ドキュメント](./07-API-ドキュメント.md) を参照してください。
