# 🧪 08. テスト戦略

## 🛠️ 検証ワークフロー (Verification Workflow)

本プロジェクトでは、「エレガントなビジュアル」と「スムーズな操作感」を維持するため、開発サイクルにおいて以下の検証ステップを**厳守**します。

### 1. 作業ループ

1.  **実装・修正**: 指示に基づきコードを編集する。
2.  **ローカル検証**: 修正後、必ずローカルサーバーを起動し、ブラウザで実際の画面を確認する。
3.  **自己修正**: 表示崩れやエラーがあれば、その場で修正し再検証。
4.  **報告**: 正常動作が確認できて初めてユーザーへ完了報告。

### 2. Chrome DevTools による徹底検証

ブラウザのデベロッパーツール（F12）を使用して、以下の項目を確認します。

- **レスポンシブ表示**:
  - モバイル (iPhone SE/12 Pro/Pixel 5 等)
  - タブレット (iPad Air)
  - デスクトップ
- **コンソールエラー**:
  - 赤い文字のエラーが出ていないか、JavaScript が正しくロードされているか。
- **ネットワークパフォーマンス**:
  - 画像の読み込みが重すぎないか（LCP 指標の確認）。

## 📝 チェックリスト

| 項目               | 確認内容                                                               |
| :----------------- | :--------------------------------------------------------------------- |
| **レイアウト**     | PC/スマホ両方で要素が重なったり隠れたりしていないか                    |
| **リンク**         | 全てのアンカー、ボタンが正しい遷移先（またはセクション）を指しているか |
| **アニメーション** | フェードインやスクロール制御がスムーズに動作するか                     |
| **フォーム**       | 問い合わせフォームがバリデーションを含め正しく動作するか               |
| **ロゴ**           | ローディング画面、ヘッダー、フッターで正しく表示されているか           |

## ⚠️ 注意事項

既存の「エレガントさ」を損なう修正（例：彩度の高すぎる色の使用、粗い画像の追加）は絶対に行わないでください。

詳細は [デプロイメント](./09-デプロイメント.md) を参照してください。
